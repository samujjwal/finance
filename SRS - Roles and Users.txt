Software Requirements Specification (SRS)
Module: User Role Management & User Management
1. User Role Management
1.1 Overview
User Role Management enables creation, modification, approval, and lifecycle management of roles. Roles are linked to functions and user types, which are ultimately assigned to users.
* A role → contains functions
* A user → can have multiple roles
* Roles → must be approved before use
1.2 Create User Role
Requirement ID: RS-IA-RM-00-01-1
Description
Authorized users can create roles and assign functions. Approval is required before use.
Stimulus/Response
* User creates role
* System saves and sends for approval
Functional Requirements
* Input fields:
   * User Role ID (mandatory, unique, max 8, alphanumeric)
   * User Role Name (mandatory)
   * User Type (mandatory dropdown)
   * Status = Pending Approval
* System behavior:
   * Valid → Create role (Pending Approval)
   * Invalid → Error message
Exclusions
None

2. User Management
2.1 Overview
User Management handles the full lifecycle of users with a dual control (maker-checker) mechanism.
* Users linked to:
   * Branch
   * Roles
   * User Types
* Approval required for most actions
2.2 Create User
Requirement ID: RS-IA-UM-01-01-1
Functional Requirements
Mandatory fields:
* Branch ID
* User ID (unique, max 15)
* Password (max 10, confirm required)
* First Name, Surname
* Designation
* Telephone
Optional:
* Email
* Extension
* Mobile
User Type: dropdown
System Behavior
* Valid → Pending Approval
* Invalid → Error
