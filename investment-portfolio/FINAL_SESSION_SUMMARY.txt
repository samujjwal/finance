================================================================================
KEY TECHNOLOGY INSIGHTS
================================================================================

NestJS + Prisma Architecture:
  - Service-layer pattern with dependency injection
  - Decorators for access control (@ModuleAccess)
  - Guards for authorization (ModuleAccessGuard)
  - DTOs for input validation
  - Prisma transactions for atomicity
  - Type-safe database queries

Double-Entry Accounting Pattern:
  - JournalEntry contains multiple JournalLines
  - Each line has debitAmount and creditAmount
  - Validation: sum(debit) = sum(credit) ±0.005
  - LedgerAccount tracks balance (debit-normal vs credit-normal)
  - Bridge services create GL entries automatically

Multi-Tenant Design:
  - Organization.modules flags (featurization)
  - @ModuleAccess decorator filters by module
  - All queries filtered by organizationId
  - Data isolation per organization
  - Audit trail tracks user and timestamp

Nepal Localization:
  - BS calendar with holiday definitions
  - VAT inclusive/exclusive calculation
  - TDS with thresholds and investor types
  - IRD export format compliance
  - Fiscal year as "YYYY/YY" string
